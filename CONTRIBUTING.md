# Contributing

## Branching Strategy
- **main**: Production-ready code.
- **develop**: Integration branch for current development.
- **feature/<short-name>**: Feature branches.

## Pull Request Checklist
- [ ] Changed files list
- [ ] Run instructions
- [ ] Test output (pasted)
- [ ] At least one unit test (if logic added)

## Validation
- Ensure `yarn start` works.
- Ensure `yarn test` passes.
- Do not merge if simulator run fails.
